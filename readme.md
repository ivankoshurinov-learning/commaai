# Курсовой проект OTUS ML Basic

## О чём проект

Проект выполнен в рамках курсовой работы по курсу OTUS «ML Basic»
(курсовое задание — Jupyter Notebook + презентация).

Идея: взять логи телеметрии реальных проездов машины (скорость, угол руля, ускорения и т.п.) и по этим данным научиться определять, что сейчас делает машина — едет прямо или поворачивает налево/направо.

## Датасет

Я использую открытый датасет **comma.ai driving dataset (2016)** с сайта Internet Archive (https://archive.org/details/comma-dataset), который упомянут в списке рекомендованных источников данных для работ. Кроме того, это сам по себе один из самых известных датасетов и проектов в области автономного вождения автомобиля.

Датасет содержит папку `camera/` с видео, но я использую только числовую телеметрию из папки `log/` с файлами `*.h5`.
Несмотря на то, что я ранее не сталкивался с этим форматом, его удобно напрямую читать через `h5py` и `pandas`, превращая в DataFrame, с которым мы знакомы по занятиям

## Что я хочу сделать

Формулировка задачи:

- **Тип задачи**: многоклассовая классификация.
- **Признаки**:
  - скорость (`speed`);
  - угол поворота руля (`steering_angle`);
  - ускорения (`accel_x`, `accel_y`, `accel_z`);
  - данные гироскопа (`gyro_*`);
- **Цель**: по этим признакам предсказывать класс манёвра:
  - `straight` — едем прямо;
  - `turn_left` — поворачиваем налево;
  - `turn_right` — поворачиваем направо.

Целевую переменную `maneuver_class` я планирую построить из `steering_angle` с помощью порогов (например, большие положительные углы — поворот направо, большие отрицательные — налево, маленькие — прямо).

## План по шагам

Структура ноутбуков:

1. `01_data_loading.ipynb`
   - подключить данные из `log/*.h5`;
   - посмотреть структуру файлов;
   - сделать первый EDA: описательные статистики, пропуски, распределения.

2. `02_preprocessing.ipynb`
   - очистить данные (фильтры по физическим ограничениям, шум, выбросы);
   - сформировать целевую переменную `maneuver_class`;
   - разделить данные на train / validation / test (по поездкам, чтобы не было утечки).

3. `03_modeling.ipynb`
   - baseline‑модель (DummyClassifier);
   - логистическая регрессия;
   - RandomForest;
   - сравнение метрик, попытка оценить и побороть переобучение.

## Как запускать

1. Установить зависимости (см. ниже).
2. Запустить Jupyter Notebook.
3. Открывать ноутбуки в таком порядке:
   - `notebooks/01_data_loading.ipynb`
   - `notebooks/02_preprocessing.ipynb`
   - `notebooks/03_modeling.ipynb`
